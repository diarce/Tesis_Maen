Paso 1

Prerrequisitos

Verificá que tu entorno cumple estos requisitos antes de comenzar.

🐍
Python 3.10 o superior Requerido Verificá con python --version o python3 --version. Descargá desde python.org/downloads.
📦
pip (gestor de paquetes) Requerido Incluido en Python 3.4+. Verificá con pip --version.
🌐
Conexión a internet Requerido (instalación) Necesaria para instalar paquetes. El modo Demo funciona sin internet.
🗜
Descompresor ZIP Requerido Windows 10+ incluye soporte nativo. macOS/Linux: comando unzip.
🎭
Playwright (Chromium) Opcional Solo necesario para auditar sitios con JavaScript dinámico (campo "dinámico" = Sí en la app). Se instala en un paso adicional.
💡
¿No tenés Python instalado?

En Windows, descargá el instalador desde python.org y marcá la opción "Add Python to PATH" durante la instalación. En macOS/Linux, recomendamos usar Homebrew (brew install python) o el gestor de paquetes del sistema.

Paso 2 — Opciones de instalación

Instalación local

Seleccioná tu sistema operativo y seguí los pasos en orden.

1
Descargar y descomprimir el proyecto

Descargá el archivo auditoria_mayoristas_v3.zip y descomprimilo haciendo clic derecho → Extraer todo. Abrí una terminal (Win+R → cmd o buscá "PowerShell") y navegá a la carpeta:

PowerShell / CMD 
PS C:\Users\tuusuario> cd C:\ruta\a\majorista_audit
# Si descomprimiste en Descargas:
PS C:\Users\tuusuario> cd %USERPROFILE%\Downloads\majorista_audit
2
Verificar versión de Python
PowerShell
PS> python --version
Python 3.12.3   # debe ser 3.10 o mayor
3
Crear entorno virtual

Un entorno virtual aísla las dependencias del proyecto y evita conflictos con otros programas Python.

PowerShell
PS> python -m venv .venv
PS> .venv\Scripts\activate
(.venv) PS>   # el prefijo confirma que el entorno está activo
⚠

Si PowerShell bloquea la ejecución de scripts, ejecutá primero: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

4
Instalar dependencias
PowerShell
(.venv) PS> pip install -r requirements.txt
# Esperar a que finalice (1–3 minutos según la conexión)
Successfully installed streamlit-1.58.0 beautifulsoup4-4.14.3 ...
5
(Opcional) Instalar Playwright para sitios dinámicos

Solo si vas a auditar sitios con JavaScript dinámico (campo "Dinámico" activado en la app).

PowerShell
(.venv) PS> playwright install chromium
Downloading Chromium 131.0.6778.108 ...

Iniciar la aplicación
PowerShell
(.venv) PS> streamlit run app.py

Paso 3 — Alternativa en la nube

Despliegue en Streamlit Community Cloud

Publicá la aplicación en internet de forma gratuita, sin mantener un servidor propio.

☁️
¿Cuándo usar Streamlit Cloud?

Ideal para compartir los informes y el dashboard con colegas o directores de tesis. La base de datos SQLite se reinicia al despertar la app (free tier), por lo que se recomienda usar el modo Demo o generar los datos localmente y subir solo los CSV.

📁
1 — Subir el proyecto a GitHub

Creá un repositorio GitHub (puede ser privado en plan gratuito de GitHub) y subí los archivos del proyecto.

$ git init
$ git add .
$ git commit -m "AuditMayorista v3 - producción"
$ git remote add origin https://github.com/tu-usuario/auditmayor.git
$ git push -u origin main

Paso 5

Resolución de problemas frecuentes

Hacé clic en cada problema para ver la solución.

❌ streamlit: command not found o 'streamlit' no se reconoce

El entorno virtual no está activo o Streamlit no se instaló correctamente.

Terminal
# Activar el entorno virtual primero:
# Windows:
PS> .venv\Scripts\activate
# macOS/Linux:
$ source .venv/bin/activate
# Luego reinstalar:
(.venv) $ pip install streamlit
❌ ModuleNotFoundError: No module named 'bs4' (u otro módulo)

Las dependencias no se instalaron correctamente, o se instalaron en el entorno global en lugar del virtual.

Terminal
# Verificar que el entorno virtual esté activo (debe aparecer (.venv))
(.venv) $ pip install -r requirements.txt --force-reinstall
❌ PermissionError al ejecutar scripts en Windows

PowerShell bloquea la ejecución de scripts por política de seguridad.

PowerShell (como Administrador)
PS> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
# Responder 'S' y luego reintentar la activación del entorno
❌ La app abre pero no muestra datos después de la Demo

La base de datos puede estar vacía o corrupta. Limpiala y volvé a correr la Demo.

Terminal
(.venv) $ make clean     # borra la BD y los outputs
(.venv) $ make demo      # regenera datos frescos
# Luego en la app: pestaña Resultados → refrescar
❌ Error al auditar un sitio: robots.txt bloqueado

El archivo robots.txt del sitio niega el acceso a bots. El sistema lo respeta por protocolo ético. En el log verás: [ÉTICA] Acceso NO permitido por robots.txt.

Opciones: (a) excluir ese sitio de la muestra y documentar el motivo en el informe, (b) verificar manualmente el robots.txt del sitio para entender las restricciones, (c) contactar al responsable del sitio para solicitar acceso autorizado para investigación.

❌ playwright install falla en Linux/servidor

Faltan dependencias del sistema operativo necesarias para Chromium.

Bash — Ubuntu/Debian
$ sudo apt-get install -y libgbm-dev libasound2 libatk1.0-0 \
    libatk-bridge2.0-0 libcups2 libdbus-1-3 libdrm2 \
    libxcomposite1 libxdamage1 libxfixes3 libxrandr2 libxss1
$ playwright install-deps chromium
❌ En Streamlit Cloud: la app genera error al importar módulos

Verificá que el archivo requirements.txt esté en la raíz del repositorio y que todos los módulos del proyecto estén incluidos en el commit. El directorio modules/ debe contener el archivo __init__.py.

Estructura esperada en GitHub
tu-repo/
├── app.py
├── config.py
├── main.py
├── requirements.txt
├── data/
